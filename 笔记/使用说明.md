# LLaVA-PruMerge 使用说明

本说明将从零开始，带你一步步在本地环境中跑通本项目（以推理/评测为主，训练参考官方文档）。

> 注意：以下步骤假设你已经基本会使用 Conda、CUDA 驱动和命令行。如果环境有差异，可按提示调整路径或命令。

---

## 一、创建项目环境

### 1. 克隆项目代码

```bash
git clone https://github.com/42Shawn/LLaVA-PruMerge.git
cd LLaVA-PruMerge
```

如果你现在看到这个说明，说明你已经在仓库目录下了，只需要确认当前路径正确：

```bash
pwd
ls
```

应能看到 `llava/`、`scripts/`、`docs/` 等目录。

### 2. 准备 Python 虚拟环境

建议使用 Conda 创建独立环境（Python 3.10 左右版本通常兼容性最好）：

```bash
conda create -n llava-prumerge python=3.10 -y
conda activate llava-prumerge
```

如果使用 `virtualenv` 或 `pyenv`，也可以，保持 Python 3.9–3.11 之间即可。

### 3. 安装依赖

项目使用 `pyproject.toml` 管理依赖，你可以用 `pip` 直接安装：

```bash
pip install -e .
```

如果网络较慢，建议先配置好国内源（如清华/阿里），或者在有代理的环境中安装。

安装完成后，可以简单校验一下：

```bash
python -c "import llava; print(llava.__file__)"
```

若没有报错，说明基础依赖安装成功。

## 二、准备与 LLaVA-1.5 一致的环境

项目在 `README.md` 中要求：“Step.0: Set the environment the same as LLaVA-1.5”。

这主要包含：

- **CUDA 与 GPU 驱动**：建议使用 CUDA 11.x / 12.x 和对应的 PyTorch 版本（与 LLaVA-1.5 官方推荐一致）。
- **PyTorch + CUDA**：例如：

```bash
pip install "torch==2.1.0" "torchvision==0.16.0" --index-url https://download.pytorch.org/whl/cu118
```

> 具体版本请参考你本机 CUDA 版本和 LLaVA 文档，确保 PyTorch 和 CUDA 匹配。

如果你之前已经成功跑通 LLaVA-1.5，那么直接在同样的环境下安装本项目即可。

## 三、下载模型与权重（推理）

### 1. 下载 PruMerge LoRA 权重

根据 `README.md` 说明：

- 从作者 Hugging Face 主页下载 LoRA 版权重：
  - 链接见 `README.md`：`https://huggingface.co/yuzhang`
- 下载后，将权重放到：

```text
checkpoints/llava-v1.5-7b-lora-prunemerge
```

如果目录不存在，先创建：

```bash
mkdir -p checkpoints/llava-v1.5-7b-lora-prunemerge
```

> 具体文件结构通常与 LLaVA-1.5 LoRA 权重一致，例如包含 `adapter_model.bin` / `adapter_config.json` 等。

### 2. 底座 LLaVA/LLaMA 模型 —— 详细步骤

PruMerge 基于 **LLaVA-1.5**（背后是 **Vicuna/LLaMA**）。这一部分的目标是：

- 准备 **语言模型基座**（Vicuna / LLaMA）；
- 准备 **LLaVA-1.5** 的合并权重或 LoRA 版本；
- 让这些路径和文件名与 `scripts/v1_5/*` 中脚本的参数保持一致。

下面给出一个**推荐且具体的做法**（以 Vicuna-7B + LLaVA-v1.5-7B 为例）：

#### 2.1 准备 Vicuna / LLaMA 基座模型

1. 去相应的模型主页下载基座模型（这里假设你已经按 Vicuna/LLaMA 官方流程拿到了权重）；
2. 将基座模型放在一个固定目录，例如：

```text
/data/models/vicuna-7b-v1.5/
```

目录内通常包含：

- `config.json`
- `generation_config.json`（如果有）
- `pytorch_model-*.bin` 或 `model-*.safetensors`
- `tokenizer.model` / `tokenizer_config.json` / `special_tokens_map.json` 等

> 这一部分和普通使用 Vicuna/LLaMA 的方法一样，不依赖 PruMerge，**只要能被 Hugging Face Transformers 正常加载即可**。

#### 2.2 准备原始 LLaVA-1.5 模型（可选但推荐）

LLaVA-1.5 本身也是在 Vicuna/LLaMA 上加上视觉 encoder + projector 之后再对齐训练得到的。你有两种思路：

- **思路 A：直接使用官方 LLaVA-1.5 模型作为基线**  
  - 从 `liuhaotian/llava-v1.5-7b`（或类似名称）下载完整 LLaVA-1.5 模型；
  - 放到例如：

```text
/data/models/llava-v1.5-7b/
```

  - 然后在 PruMerge 的脚本中，把 `--model-path` 指向这个目录，再额外加载 PruMerge 的 LoRA。

- **思路 B：仅使用 LLaVA-1.5 的 LoRA**（更接近本项目推荐方式）  
  - 使用原始 Vicuna/LLaMA 作为 `--model-path`；
  - 使用 LLaVA-1.5 的 LoRA（如官方 `llava-v1.5-7b-lora`）作为 `--lora-path`；
  - 再叠加 PruMerge 的 LoRA。

> 不同脚本写法略有差异，但核心思想是：**“基座 LLM + LLaVA 对齐（可由 LoRA 提供）+ PruMerge LoRA”**。

#### 2.3 准备 PruMerge 的 LoRA 权重（你当前项目对应的）

前面已经说明，你可以从 Hugging Face 下载：

- `yuzhang/llava-prumerge-vicuna-7b-v1.5-lora`

推荐放在本项目根目录下：

```text
LLaVA-PruMerge/
  checkpoints/
    llava-v1.5-7b-lora-prunemerge/   ← 这里放 PruMerge LoRA
```

你可以用 `git lfs` 或 `huggingface-cli` 下载，例如：

```bash
cd /path/to/LLaVA-PruMerge
mkdir -p checkpoints/llava-v1.5-7b-lora-prunemerge

huggingface-cli download \
  yuzhang/llava-prumerge-vicuna-7b-v1.5-lora \
  --local-dir checkpoints/llava-v1.5-7b-lora-prunemerge \
  --local-dir-use-symlinks False
```

#### 2.4 让脚本能同时看到“基座 + LoRA”

以评测脚本为例（伪代码风格，但结构与 `scripts/v1_5/eval/textvqa.sh` 类似）：

```bash
CUDA_VISIBLE_DEVICES=0 python -m llava.eval.eval_textvqa \
  --model-path /data/models/vicuna-7b-v1.5 \        # 底座 LLaMA/Vicuna
  --llava-lora-path /data/models/llava-v1.5-7b-lora \  # 原始 LLaVA-1.5 LoRA（如果脚本有这个参数）
  --prumerge-lora-path checkpoints/llava-v1.5-7b-lora-prunemerge \  # PruMerge LoRA
  --other-args ...
```

实际项目中，参数名可能是：

- `--model-path`
- `--model-base`
- `--lora-path`
- `--mm-projector-type` 等

你需要：

1. 打开具体的 `scripts/v1_5/eval/*.sh`；
2. 找到类似 `--model-path` / `--lora-path` 的地方；
3. 把它们分别改成你上面准备好的目录，例如：

```bash
--model-path /data/models/vicuna-7b-v1.5
--lora-path checkpoints/llava-v1.5-7b-lora-prunemerge
```

如果脚本里已经写死了某个默认路径，你只要改成自己的路径即可，不影响逻辑。

#### 2.5 一个“从零开始”的最小示例配置（汇总）

假设你整合后的布局如下：

```text
/data/models/
  vicuna-7b-v1.5/             # 底座 LLaMA/Vicuna
  llava-v1.5-7b-lora/         # 原始 LLaVA-1.5 LoRA（可选）

/path/to/LLaVA-PruMerge/
  checkpoints/
    llava-v1.5-7b-lora-prunemerge/   # PruMerge LoRA
```

那你可以把某个评测脚本改成（示意）：

```bash
CUDA_VISIBLE_DEVICES=0 \
python -m llava.eval.eval_textvqa \
  --model-path /data/models/vicuna-7b-v1.5 \
  --lora-path checkpoints/llava-v1.5-7b-lora-prunemerge \
  --other-args ...
```

或者如果脚本支持单独指定 LLaVA-1.5 对齐 LoRA：

```bash
--model-path /data/models/vicuna-7b-v1.5 \
--llava-lora-path /data/models/llava-v1.5-7b-lora \
--prumerge-lora-path checkpoints/llava-v1.5-7b-lora-prunemerge
```

> 具体参数名以你实际的脚本为准，但**思路都是：基座模型在一个目录；LLaVA-1.5 / PruMerge 的 LoRA 各在一个目录；脚本参数分别指向它们**。

如果你把某个你想用的 `scripts/v1_5/eval/*.sh` 整个内容发给我，我可以基于你真实的路径帮你改成一个“可直接复制运行”的版本，并补充到本 `使用说明.md` 里作为模板。

## 四、选择并配置 PruMerge 模式

PruMerge 的核心逻辑在：

- `llava/model/multimodal_encoder/clip_encoder.py`

根据 `README.md` 提示：

- **核心模块代码**在 `clip_encoder.py` 的约第 85 行；
- **选择 PruMerge 或 PruMerge+ 方式**在约第 295 行附近的调用处。

如果你想切换不同的裁剪策略（如 PruMerge / PruMerge+），可以在相应函数内部调整：

- 例如，切换调用的函数名，或修改配置参数（保留 token 比例、阈值等）。

> 建议先使用默认配置，确认可以正常跑通，再尝试修改参数体验不同压缩率的效果。

## 五、运行推理与评测脚本

项目提供了一系列 `scripts/v1_5/eval/*.sh` 脚本，用于在不同数据集上做评测。以 `TextVQA` 为例：

### 1. 准备数据集

以 `TextVQA` 为例，你需要：

- 下载 `TextVQA` 数据集到某个路径（例如 `/data/textvqa/`）；
- 按照 LLaVA 官方预处理脚本，将其转换为 LLaVA 期望的 JSON/COCO 格式（参考原 LLaVA 的 `docs/Data.md` 与脚本）。

本仓库也有数据相关文档：

- `docs/Data.md`：描述通用数据准备方式；
- `docs/ScienceQA.md` 等：对应特定数据集的说明。

### 2. 运行 TextVQA 评测脚本

`README.md` 中提供了示例命令：

```bash
CUDA_VISIBLE_DEVICES=7 XDG_CACHE_HOME='/data/shangyuzhang/' bash scripts/v1_5/eval/testvqa.sh
```

你需要根据自己环境修改：

- `CUDA_VISIBLE_DEVICES=7` 改为你自己的 GPU 编号，如只用单卡：

```bash
CUDA_VISIBLE_DEVICES=0 XDG_CACHE_HOME='/你的缓存目录' bash scripts/v1_5/eval/testvqa.sh
```

- `XDG_CACHE_HOME` 可以设置为任何有足够空间的目录，用于存放缓存（如模型下载、特征等）。

`scripts/v1_5/eval/testvqa.sh` 内部通常会：

- 指定模型路径（基座模型 + LoRA）；
- 指定数据路径；
- 指定 batch size、图像分辨率等参数；
- 调用 `llava/eval/eval_textvqa.py` 做实际评测。

你可以打开该脚本，按需修改：

- 模型路径（如 `--model-path` / `--lora-path`）；
- 数据路径（如 `--data-path` / `--question-file` / `--annotation-file`）；
- 输出结果保存路径。

### 3. 其他评测脚本

在 `scripts/v1_5/eval/` 中，还包含例如：

- `vqav2.sh`
- `gqa.sh`
- `mme.sh`
- `mmvet.sh`
- `sqa.sh`
- `pope.sh`
- `efficient_sqa.sh`
- 等等

它们对应 `llava/eval/` 目录下的不同评测脚本（如 `eval_textvqa.py`、`eval_gqa.py`、`eval_pope.py` 等）。使用方法与上面类似：

1. 准备对应数据集；
2. 对照脚本里的参数，修改数据路径与模型路径；
3. 使用 `bash scripts/v1_5/eval/xxx.sh` 运行。

## 八、常见问题与排查方向

- **ImportError / ModuleNotFoundError**  
  - 检查是否在 `llava-prumerge` 虚拟环境中；
  - 确认 `pip install -e .` 是否成功执行。

- **CUDA 相关错误（如 `CUDA error: invalid device ordinal`）**  
  - 检查 `CUDA_VISIBLE_DEVICES` 设置是否超过实际 GPU 数量；
  - 使用 `nvidia-smi` 查看 GPU 是否可见。

- **显存不足（OOM）**  
  - 降低 batch size；
  - 降低图像分辨率；
  - 启用更激进的 PruMerge 配置（更高的裁剪率）。

- **找不到数据文件 / JSON 路径错误**  
  - 打开对应的 `scripts/v1_5/eval/*.sh`，逐个确认 `--xxx-path` 是否与你本地目录一致；
  - 根据 `docs/Data.md` 和数据集官方说明，检查是否预处理正确。

- **TextVQA：`FileNotFoundError: ... answers/llava-v1.5-7b.jsonl`**  
  - 该文件是**第一步推理**生成的，不是下载的。必须先跑完 `textvqa.sh` 里的**第一条命令**（`model_vqa_loader`），生成 `answers/llava-v1.5-7b.jsonl` 后，再跑第二条（`eval_textvqa`）才会成功。
  - 确保已按 `docs/Evaluation.md` 准备好 TextVQA 数据，并执行：`mkdir -p ./playground/data/eval/textvqa/answers`。

- **NumPy 报错：`A module that was compiled using NumPy 1.x cannot be run in NumPy 2.x`**  
  - 当前环境是 NumPy 2.x，而 bitsandbytes 等库是按 NumPy 1.x 编译的。  
  - 解决：在环境里执行 `pip install "numpy<2"` 降级到 1.x，再重新跑脚本。

- **`HFValidationError: Repo id must be in the form 'repo_name' or 'namespace/repo_name': '/data/.../llava-v1.5-7b-lora-...'`**  
  - 脚本里 `--model-path` 填的是**本地路径**，旧版 Hugging Face 会把它当 repo id 校验导致报错。  
  - 本项目已在 `llava/model/builder.py` 中改为：当 `model_path` 为本地目录时使用 `local_files_only=True` 加载，拉取最新代码后即可。  
  - 同时确认该目录下存在 `config.json`（LLaVA LoRA 一般都有）；若只有 `adapter_config.json` 没有 `config.json`，需改用「完整 LLaVA 模型路径」作为 `--model-path`，或从 HF 重新下载带 config 的 LoRA。

## 九、推荐的最小跑通流程（给你一个最短 checklist）

1. 创建并激活 Conda 环境：`conda create -n llava-prumerge python=3.10` + `conda activate llava-prumerge`  
2. 安装 PyTorch（与你 CUDA 匹配的版本）。  
3. 在项目根目录执行：`pip install -e .`  
4. 按照 LLaVA-1.5 官方文档准备好基础模型和环境。  
5. 从 Hugging Face 下载 PruMerge LoRA 权重到 `checkpoints/llava-v1.5-7b-lora-prunemerge/`。  
6. 准备一个数据集（如 TextVQA），确认数据路径。  
7. 修改 `scripts/v1_5/eval/testvqa.sh` 中的模型路径与数据路径。  
8. 执行：`CUDA_VISIBLE_DEVICES=0 bash scripts/v1_5/eval/testvqa.sh`，等待评测完成。  
9. 查看输出结果（通常是 JSON/日志），确认模型可以正常运行。
