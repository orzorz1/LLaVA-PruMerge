# 使用与训练指南

本文件只讲「怎么用」：**推理 / 评测 / 训练**。环境和数据下载请看 `环境与资源配置.md`。

---

## 一、推理与评测

### 1. 只用原始 LLaVA 推理（不带 PruMerge）

适合先确认环境与数据是否正常。

以 TextVQA 为例：

```bash
python -m llava.eval.model_vqa_loader \
  --model-path /data/models/llava-v1.5-7b \
  --question-file ./playground/data/eval/textvqa/llava_textvqa_val_v051_ocr.jsonl \
  --image-folder ./playground/data/eval/textvqa/train_images \
  --answers-file ./playground/data/eval/textvqa/answers/llava-v1.5-7b.jsonl \
  --temperature 0 \
  --conv-mode vicuna_v1

python -m llava.eval.eval_textvqa \
  --annotation-file ./playground/data/eval/textvqa/TextVQA_0.5.1_val.json \
  --result-file ./playground/data/eval/textvqa/answers/llava-v1.5-7b.jsonl
```

关键点：

- `--model-path` 直接指向完整 LLaVA 模型（如 `liuhaotian/llava-v1.5-7b` 的本地目录），不再拆成 base + LoRA。
- 第一条命令负责写出答案 jsonl，第二条命令负责用官方标注做评测。

### 2. 使用 PruMerge LoRA 推理（当前推荐方式）

仍以 TextVQA 为例，使用 `scripts/v1_5/eval/textvqa.sh`：

```bash
CUDA_VISIBLE_DEVICES=0 bash scripts/v1_5/eval/textvqa.sh
```

其中脚本内容应类似：

```bash
python -m llava.eval.model_vqa_loader \
    --model-base lmsys/vicuna-7b-v1.5 \
    --model-path checkpoints/llava-v1.5-7b-lora-prunemerge \
    --question-file ./playground/data/eval/textvqa/llava_textvqa_val_v051_ocr.jsonl \
    --image-folder ./playground/data/eval/textvqa/train_images \
    --answers-file ./playground/data/eval/textvqa/answers/llava-v1.5-7b.jsonl \
    --temperature 0 \
    --conv-mode vicuna_v1

python -m llava.eval.eval_textvqa \
    --annotation-file ./playground/data/eval/textvqa/TextVQA_0.5.1_val.json \
    --result-file ./playground/data/eval/textvqa/answers/llava-v1.5-7b.jsonl
```

含义：

- `--model-base`：基座语言模型（Vicuna-7B）。  
- `--model-path`：PruMerge 的 LoRA 目录。  
- `builder.py` 会用 base + LoRA 组合出一个带 PruMerge 的 LLaVA 模型。

> 想切到 PruMerge+，只需要把 `--model-path` 换成你下载的 plus 版本目录即可。

### 3. 其它评测脚本的用法

所有 `scripts/v1_5/eval/*.sh` 结构基本相同：

- 第一条 `python -m llava.eval.model_vqa_loader ...` 负责生成答案 jsonl。  
- 第二条（如果有）负责用官方脚本评测指标。  

只要你把这几个参数改成自己的路径即可：

- `--model-path` / `--model-base` / `--lora-path`（模型 + LoRA）  
- `--question-file`（eval.zip 里通常已经给好）  
- `--image-folder`（你下载好的图像目录）  
- `--answers-file`（写出预测结果的路径）  

其它数据集（VQAv2、GQA、MME、MMBench 等）的数据准备路径在 `环境与资源配置.md` 里已经列出，对应的脚本名分别是：

- `scripts/v1_5/eval/vqav2.sh`
- `scripts/v1_5/eval/gqa.sh`
- `scripts/v1_5/eval/mme.sh`
- `scripts/v1_5/eval/mmbench.sh` / `mmbench_cn.sh`
- `scripts/v1_5/eval/pope.sh`
- `scripts/v1_5/eval/sqa.sh` / `efficient_sqa.sh`
- `scripts/v1_5/eval/mmvet.sh`
- `scripts/v1_5/eval/qbench.sh` / `qbench_zh.sh`

## 二、交互式使用（Gradio / CLI）

### 1. Gradio Web Demo

在单机上跑一个简单的网页 Demo（命令示意）：

```bash
python -m llava.serve.gradio_web_server \
  --model-path /data/models/llava-v1.5-7b \
  --lora-path checkpoints/llava-v1.5-7b-lora-prunemerge \
  --conv-template llava_v1
```

说明：

- 这里以“完整 LLaVA + PruMerge LoRA”为例，你也可以只用原始 LLaVA（去掉 `--lora-path`）。  
- `--conv-template` 需要和模型版本匹配（常见：`llava_v1`、`vicuna_v1`），可参考 LLaVA 原仓库或 `llava/conversation.py`。

### 2. 命令行 CLI

可以用 `llava/serve/cli.py` 做简单命令行对话（示意）：

```bash
python -m llava.serve.cli \
  --model-path /data/models/llava-v1.5-7b \
  --lora-path checkpoints/llava-v1.5-7b-lora-prunemerge
```

## 三、训练与微调（概览）

LLaVA（以及本项目）的大致训练流程分三类场景：

1. **从 Vicuna + 图像预训练 + 指令微调，训练出原始 LLaVA。**  
2. **在原始 LLaVA 上，用 LoRA 做任务微调（task LoRA）。**  
3. **在本项目代码下，用 PruMerge 的 CLIP 编码器做 LoRA 微调，得到 PruMerge LoRA。**

下面给一个「你能在本项目里照抄的」简化版说明。

### 1. 使用原始 LLaVA 做任务微调（不涉及 PruMerge）

推荐用 LLaVA 官方的 task LoRA 脚本：

- LoRA 版本：`scripts/v1_5/finetune_task_lora.sh`  
- 全参数版本：`scripts/v1_5/finetune_task.sh`

以 `finetune_task_lora.sh` 为例（13B，示意）：

```bash
deepspeed llava/train/train_mem.py \
  --lora_enable True --lora_r 128 --lora_alpha 256 --mm_projector_lr 2e-5 \
  --deepspeed ./scripts/zero3.json \
  --model_name_or_path liuhaotian/llava-v1.5-13b \
  --version v1 \
  --data_path ./playground/data/llava_v1_5_mix665k.json \
  --image_folder ./playground/data \
  --vision_tower openai/clip-vit-large-patch14-336 \
  --mm_projector_type mlp2x_gelu \
  --mm_vision_select_layer -2 \
  --mm_use_im_start_end False \
  --mm_use_im_patch_token False \
  --output_dir ./checkpoints/llava-v1.5-13b-task-lora \
  ...
```

如果你要用**自己的数据**，只需：

- 按 `docs/Finetune_Custom_Data.md` 的格式准备 `data_path` 指向的 json；  
- 把 `--data_path` / `--image_folder` 改成自己的路径；  
- 其它超参（batch size / lr / epoch）按显存调整。

### 2. 使用 PruMerge 代码做 LoRA 微调（复现/改进本项目）

本项目自带了一些 PruMerge 训练脚本（例如 7B 的 `scripts/v1_5/finetune_lora_7b.sh`）：

```bash
deepspeed llava/train/train_mem.py \
  --lora_enable True --lora_r 128 --lora_alpha 256 --mm_projector_lr 2e-5 \
  --deepspeed ./scripts/zero3.json \
  --model_name_or_path lmsys/vicuna-7b-v1.5 \
  --version v1 \
  --data_path ./playground/data/llava_v1_5_mix665k.json \
  --image_folder ./playground/data \
  --vision_tower openai/clip-vit-large-patch14-336 \
  --pretrain_mm_mlp_adapter ./checkpoints/llava-v1.5-7b-pretrain/mm_projector.bin \
  --mm_projector_type mlp2x_gelu \
  --mm_vision_select_layer -2 \
  --mm_use_im_start_end False \
  --mm_use_im_patch_token False \
  --output_dir ./checkpoints/llava-v1.5-7b-lora-prumerge-my \
  ...
```

要点：

- 训练入口仍然是 `llava/train/train_mem.py`，多模态结构由 `llava/model/multimodal_encoder/clip_encoder.py` 决定——这里已经替换成了 PruMerge 版 CLIP。  
- `--model_name_or_path` 是 Vicuna 基座；  
- `--pretrain_mm_mlp_adapter` 是多模态 projector 的初始化（一般用官方预训练好的 `mm_projector.bin`）；  
- 训练产出的 LoRA 就是类似 `yuzhang/llava-prumerge-vicuna-7b-v1.5-lora` 这样的权重。

你要做的只是：

1. 按 `docs/Data.md` 准备好训练数据和图片；  
2. 把 `data_path`、`image_folder`、`output_dir` 改成自己的路径；  
3. 按显存调整 batch size / epoch / lr；  
4. 运行对应的 `bash scripts/v1_5/finetune_lora_7b.sh` 或你拷出来修改过的脚本。

### 3. 从“完全原始 Vicuna”开始训练一套自己的 LLaVA + PruMerge（高级）

完整流程大致是：

1. **预训练多模态 projector**：  
   - 使用 `scripts/v1_5/pretrain.sh`，从 Vicuna-7B + CLIP + 预训练图文数据（CC3M/LAION 等）出发，训练出 `mm_projector.bin`。  
2. **在此基础上做指令微调**：  
   - 使用类似 `finetune_lora_7b.sh` 的脚本，加载上一步 `mm_projector.bin`，在指令数据上做 LoRA 微调。  
3. **整个过程都在 PruMerge 代码库中进行**：  
   - 由于 `clip_encoder.py` 已经换成 PruMerge 实现，训练得到的模型天然带 token 削减能力。

这一套训练对数据量和算力要求都比较高，更适合你已经跑通推理与简单微调之后再尝试。

## 四、推荐使用顺序

1. **先用原始 LLaVA 跑一个小任务（例如 TextVQA）**  
   - 目的：确认环境、数据路径、CUDA 都是 OK 的。  
2. **再切换到 PruMerge LoRA 做同一个任务**  
   - 直接复用同一套脚本，只改 `--model-base` / `--model-path`。  
3. **最后再考虑自己训练 LoRA 或从头训练**  
   - 先用 `finetune_task_lora.sh` 在原始 LLaVA 上做任务 LoRA；  
   - 再考虑用 `finetune_lora_7b.sh` 在 PruMerge 上做实验。  

这样你每一步都能明确“哪里出问题”，不会一上来就被环境/数据/训练同时卡住。
