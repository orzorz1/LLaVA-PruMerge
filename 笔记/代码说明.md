# LLaVA-PruMerge 代码粗略说明

本文件从「目录/文件职责」的角度，对项目做一个**结构化的、但不深入到每一行代码**的说明，帮助你快速建立整体认识。后续如果你想要针对某个模块做深入解读，可以在这个文档基础上再细化。

## 一、项目根目录大致结构

- **`llava/`**：项目的核心 Python 包，包含模型结构、训练代码、评测脚本和服务端（Gradio/多进程）等，是最重要的目录。
- **`scripts/`**：各种 Shell 脚本，主要用于数据转换、预训练、微调、评测等的命令封装。
- **`docs/`**：文档目录，详细说明数据准备、模型 Zoo、评测、平台支持（macOS/Windows/Intel）等。
- **`playground/`**：示例数据与 prompt，用于本地 Playground / Demo 等。
- **`images/`**：论文/项目相关示意图（如架构图、示例图）。
- **`predict.py`**：通常是一个较为简单的推理入口脚本，用于单次或批量预测（依赖于 `llava` 内部模块）。
- **`pyproject.toml`**：Python 项目配置与依赖管理文件。
- **`cog.yaml`**：与 Replicate/cog 等部署相关的配置文件（容器化/推理服务配置）。
- 其他如 `.devcontainer/`、`.git/` 等：开发容器配置与 Git 元数据，对理解模型功能不关键。

下面按主要目录深入说明。

## 二、`llava/` 目录（核心包）

`llava/` 是项目实现的核心，主要包含模型、训练、评测与服务模块。

### 1. 顶层文件

- **`__init__.py`**：标记 `llava` 为 Python 包，通常也会暴露一些顶层接口。
- **`constants.py`**：放一些常量配置，如特殊 token、默认 prompt 模板等。
- **`conversation.py`**：对话模板与对话格式相关逻辑，例如不同对话风格、系统提示等。
- **`utils.py`**：通用工具函数，如路径处理、日志、数据加载小工具等。
- **`mm_utils.py`**：与多模态（multi-modal）相关的工具函数，比如图像预处理、特征对齐等。

### 2. `llava/model/` —— 模型结构相关

`model/` 目录包含所有与模型结构本身相关的代码。

- **`model/__init__.py`**：导出模型构建等接口。
- **`model/builder.py`**：模型构建入口，根据配置/参数创建完整的 LLaVA 模型（语言模型 + 图像编码器 + 多模态 projector）。
- **`model/llava_arch.py`**：LLaVA 整体架构定义（如何把语言模型和视觉模块结合在一起）。
- **`model/utils.py`**：模型相关的小工具，例如加载权重、权重转换等。
- **`model/apply_delta.py` / `model/make_delta.py` / `model/consolidate.py`**：与权重“delta”合并/转换相关的工具，比如把 LoRA 或 diff 权重应用到基座模型上，或合并不同来源的权重。

#### 2.1 `model/language_model/` —— 语言模型部分

这里主要是对 LLaMA / MPT 等语言模型的封装。

- **`model/language_model/llava_llama.py`**  
  - 封装 LLaMA / Vicuna 等 LLaVA 使用的语言模型结构；
  - 一般会定义如何加载 Hugging Face 权重、如何与多模态部分交互等。

- **`model/language_model/llava_mpt.py`**  
  - 类似功能，但基于 MPT（MosaicML 的 Transformer）模型；
  - 与 LLaMA 版本的差异在于底层架构和配置。

- **`model/language_model/mpt/` 子目录**  
  - `configuration_mpt.py`：MPT 配置定义；
  - `modeling_mpt.py`：MPT 模型结构；
  - `attention.py` / `blocks.py` / `norm.py` / `param_init_fns.py`：MPT 内部模块实现（注意力、block 结构、归一化、参数初始化等）；
  - `adapt_tokenizer.py`：对 tokenizer 做适配；
  - `custom_embedding.py`：自定义 embedding 相关；
  - `flash_attn_triton.py`：使用 FlashAttention/Triton 加速注意力的实现；
  - `hf_prefixlm_converter.py`：把 Hugging Face 中的 PrefixLM 模型转换为当前实现所需格式；
  - `meta_init_context.py`：与 Meta 初始化 / 上下文相关的小工具。

> 这一块基本是“语言模型主体”，与 PruMerge 的核心创新（token 削减）相比，更偏通用 LLM 封装。

#### 2.2 `model/multimodal_encoder/` —— 图像编码 + PruMerge 核心

- **`model/multimodal_encoder/builder.py`**：创建多模态 encoder 的入口，根据配置选择 CLIP 等视觉 backbone，并与后续 projector 对接。
- **`model/multimodal_encoder/clip_encoder.py`**：**本项目的核心文件之一**：
  - 封装 CLIP 图像编码器；
  - 在此处加入 PruMerge / PruMerge+ 的 token 削减逻辑；
  - 通过对视觉 token 的选择/合并，使得在保证性能前提下减少计算量。
  - 根据 `README` 提示，约第 85 行附近为核心实现，约第 295 行附近为选择具体削减方式的调用点。

#### 2.3 `model/multimodal_projector/` —— 图像特征到语言空间

- **`model/multimodal_projector/builder.py`**：
  - 把视觉 encoder 输出的特征通过 projector（如线性层、MLP）映射到语言模型可以理解的 token 表示上；
  - 决定“视觉 token 如何插入到文本 token 序列中”这一关键步骤。

## 三、`llava/train/` 目录 —— 训练与微调逻辑

- **`train/train.py`**：主训练脚本入口（标准版本），包括参数解析、数据加载、模型初始化、训练循环、保存 checkpoint 等。
- **`train/train_xformers.py`**：使用 xFormers 加速注意力实现的训练版本（对大模型显存更友好）。
- **`train/train_mem.py`**：针对显存/内存优化的训练变体（例如 gradient checkpointing、offload 等）。
- **`train/llava_trainer.py`**：Trainer 类，封装训练流程（前向、反向、优化器更新、日志记录、分布式训练等）。
- **`train/llama_flash_attn_monkey_patch.py` / `train/llama_xformers_attn_monkey_patch.py`**：
  - 对 LLaMA 的注意力实现进行“猴子补丁”，把原本的注意力替换为 FlashAttention/Xformers 等更高效的版本。

整体来看，`llava/train/` 提供了多种训练变体，你在写训练脚本和看 `scripts/` 下的 `.sh` 文件时，会频繁看到这些 Python 文件被调用。

## 四、`llava/eval/` 目录 —— 评测与可视化

`eval/` 包含针对不同数据集或 benchmark 的评测脚本以及生成网页/表格的工具。

### 1. 通用/基类

- **`eval/model_vqa_loader.py`**：VQA 任务通用的加载器/评测框架，很多数据集评测脚本会基于此。
- **`eval/model_vqa.py`**：通用视觉问答模型评测脚本（可能接收多种配置，调用 loader）。
- **`eval/run_llava.py`**：运行 LLaVA 做推理/评测的通用接口。

### 2. 针对具体数据集的评测脚本

常见文件包括（与你在 `scripts/v1_5/eval/*.sh` 中看到的一一对应）：

- **`eval/eval_textvqa.py`**：TextVQA 数据集评测。
- **`eval/eval_science_qa.py` / `eval/eval_science_qa_gpt4.py`**：ScienceQA 数据集。
- **`eval/eval_pope.py`**：POPE 数据集。
- **`eval/model_vqa_science.py` / `eval/model_vqa_science_efficient.py`**：ScienceQA 相关的多模态 VQA 评测实现。
- **`eval/model_vqa_qbench.py`**：QBench 相关评测。
- **`eval/model_vqa_mmbench.py`**：MMBench 相关评测。
- **`eval/m4c_evaluator.py`**：与 M4C（多模态阅读理解模型）评测相关的工具。
- 还有其他为 LLaVA-Bench / GPT review 等服务的脚本：
  - `eval/eval_gpt_review.py`
  - `eval/eval_gpt_review_visual.py`
  - `eval/eval_gpt_review_bench.py`
  - `eval/qa_baseline_gpt35.py`
  - `eval/summarize_gpt_review.py`

### 3. 表格与网页相关子目录

- **`eval/table/`**：
  - `rule.json`：用于 GPT review 或评分规则；
  - `prompt.jsonl`：评测用 prompt；
  - `question.jsonl` / `caps_boxes_coco2014_val_80.jsonl` 等：评测问题与图像框信息；
  - `answer/`：存放不同模型（如 llama-13b、vicuna-13b、alpaca-13b、bard、gpt35）的回答；
  - `review/`：GPT 对比点评结果；
  - `results/`：汇总结果的 JSON。

- **`eval/webpage/`**：
  - `index.html` / `script.js` / `styles.css`：用来展示评测结果的静态网页；
  - `figures/*.png/jpg/svg`：网页中用到的图片素材。

> 这些更多是“结果展示与可视化”，与模型训练本身相对独立。

## 五、`llava/serve/` 目录 —— 服务端与 Demo

这个目录实现了在线服务与本地 Demo。

- **`serve/gradio_web_server.py`**：
  - 基于 Gradio 的 Web 界面；
  - 提供文本+图片输入，返回模型回答，方便交互式体验。

- **`serve/controller.py` / `serve/model_worker.py` / `serve/register_worker.py`**：
  - 实现分布式/多 worker 的服务架构；
  - `controller` 负责任务分发；
  - `model_worker` 实际加载模型并处理推理；
  - `register_worker` 将 worker 注册到 controller。

- **`serve/cli.py`**：
  - 命令行交互方式的前端，通过终端与模型对话。

- **`serve/test_message.py`**：
  - 测试消息收发/服务是否工作正常的小脚本。

- **`serve/examples/*.jpg`**：
  - Demo 演示用的图片样例。

## 六、`scripts/` 目录 —— 各类运行脚本

`scripts/` 主要包含 Shell 脚本，帮你把复杂的 Python 命令封装起来。

### 1. 顶层脚本

- **`pretrain.sh` / `pretrain_xformers.sh` / `v1_5/pretrain.sh`**：
  - 预训练脚本，指定大批量预训练数据、模型初始化、训练参数等。

- **`finetune.sh` / `finetune_lora.sh` / `finetune_qlora.sh` / `finetune_full_schedule.sh` / `finetune_sqa.sh`**：
  - 用于不同设置下的微调，如全参数微调、LoRA、QLoRA、针对特定任务的微调等。

- **`merge_lora_weights.sh` / `extract_mm_projector.py` 等**：
  - 用于把 LoRA 权重合并到基座模型，或提取多模态 projector 等辅助操作。

- **`convert_*.py` 系列（如 `convert_vqav2_for_submission.py`、`convert_gqa_for_eval.py`、`convert_mmbench_for_submission.py`、`convert_mmvet_for_eval.py`、`convert_vizwiz_for_submission.py` 等）**：
  - 将原始数据集格式转换为本项目/评测脚本需要的格式；
  - 一般会从官方 JSON/CSV/TSV 读入，输出为 LLaVA 适配的 JSONL 或 COCO 风格格式。

- **`sqa_eval_batch.sh` / `sqa_eval_gather.sh`**：
  - 针对 ScienceQA 等任务的多阶段评测（分批运行 / 汇总结果）。

- **`zero2.json` / `zero3.json` / `zero3_offload.json`**：
  - DeepSpeed 或 ZeRO 优化相关的配置，用于大模型分布式并行训练（显存分片、offload 到 CPU/磁盘等）。

### 2. `scripts/v1_5/` 子目录

专门针对 LLaVA-1.5 的训练/评测脚本。

- **`v1_5/finetune.sh` / `v1_5/finetune_lora.sh` / `v1_5/finetune_lora_7b.sh` / `v1_5/finetune_lora_7b_quant.sh` / `v1_5/finetune_task.sh` / `v1_5/finetune_task_lora.sh`**：
  - 针对 LLaVA-1.5 各种配置的微调脚本；
  - 包含不同模型规模（7B 等）、是否量化、是否 LoRA 的区分。

- **`v1_5/eval/*.sh`**（对应 `llava/eval/`）：
  - `textvqa.sh`：TextVQA 评测；
  - `vqav2.sh`：VQAv2 评测；
  - `gqa.sh`：GQA；
  - `mmbench.sh` / `mmbench_cn.sh`：MMBench 英文/中文；
  - `mmvet.sh`：MMVet；
  - `mme.sh`：MME；
  - `qbench.sh` / `qbench_zh.sh`：QBench 英文/中文；
  - `vizwiz.sh`：VizWiz；
  - `pope.sh`：POPE；
  - `sqa.sh` / `efficient_sqa.sh`：ScienceQA 相关；
  - `seed.sh` / `llavabench.sh` 等：Seed / LLaVA-Bench 等测试。

每个 `.sh` 脚本内部通常会：

- 设置环境变量（CUDA 设备、分布式配置等）；
- 指定模型路径（基座与 LoRA）；
- 指定数据路径；
- 调用对应的 `llava/eval/*.py`。

## 十、如何基于本说明继续深入阅读代码

如果你想系统性地“读懂”整个项目，可以按照下面的顺序：

1. **整体结构**：先看 `使用说明.md` + 本 `代码说明.md`，知道“有哪些模块”。  
2. **核心创新（PruMerge）**：  
   - 重点阅读：`llava/model/multimodal_encoder/clip_encoder.py`；  
   - 辅助阅读：对应的论文（看算法原理），再回到代码看实现细节。  
3. **多模态整体架构**：  
   - 看 `llava/model/builder.py` + `llava/model/llava_arch.py` + `llava/model/multimodal_projector/builder.py`；

4. **推理/评测流程**：  
   - 对照某个具体脚本（如 `scripts/v1_5/eval/textvqa.sh`）；

5. **训练流程**：  
   - 看某个训练脚本（如 `scripts/v1_5/finetune_lora.sh`）；  
   - 顺着入口看：`llava/train/train.py` / `train_xformers.py` → `llava/train/llava_trainer.py`。
